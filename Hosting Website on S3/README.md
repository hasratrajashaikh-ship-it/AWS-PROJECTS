<h1>Static Website Hosting Using Amazon S3</h1>

<h2>Project Overview</h2>

This project demonstrates how to host a static website using Amazon S3.
HTML and CSS files are uploaded to an S3 bucket, public access is configured, and S3 Static Website Hosting is enabled to make the website accessible through a browser.


<h2>AWS Services Used</h2>

1. Amazon S3 – Static website hosting
2. AWS IAM (via Bucket Policy) – Public read access

   
<h2>Project Workflow</h2>

1. Create an S3 bucket
2. Disable Block all public access
3. Upload static files (index.html, style.css)
4. Enable Static Website Hosting
5. Add a bucket policy for public read access
6. Access the website using the S3 website endpoint


<h2>Features</h2>

Serverless static website hosting

No EC2 or backend required

Cost-effective and simple setup

Publicly accessible website


<h2>Testing</h2>

Website accessed using S3 website endpoint

Static content loaded successfully in the browser
